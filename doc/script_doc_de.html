Kurz

Intro

Es ist gedacht, dass alle Skripte regelmäßig automatisch mit Hilfe von crontab ausgeführt werden. Dazu in crontab das Script bksystem.sh mit der Option -a (alle konfigurierten jobs) starten. Folgende jobs stehen zur Verfügung: Abrechnungszeitpunkt festlegen, Export verschiedener Rechnungsformate (simple: Verbrauch, Konsum, Liste; Pivottabelle als .csv, und als .html mit layout), Backup der Daten (products, user, sales,...), restore der Daten, Tabelle sales bereinigen (TODO). Damit sollten schon alle regelmäßigen Aufgaben erledigt sein. Bei der Installation müssen nur im Script die Pfade und geschwünschten jobs eingestellt werden

Software: sqlite3

Alle Skripte benötigen das commandline tool sqlite3, um die Datenabfragen zu stellen. Es wird immer die Shell sh. bash, csh etc. ist nicht nötig. Wietere tolls die benutzt werden: sed, getopts

master script: bksystem.sh

Funktion

In bksystem.sh sind alle Konfigurationen gespeichert. Das sind zum einen Defaultwerte für die Pfad und Dateiname. Zum anderen sind das die Funktionen, die automatisch, bzw. regelmäßig ausgeführt werden sollen. Des weiteren akzeptiert das script eine Reihe von Kommandozeilen Optionen.

Konfiguration

Alle Konfigurationen lassen sich in direkt in die Datei bksystem.sh schreiben. Am Anfang steht der Hilfetext. Direkt danach und kenntlich gemacht mit weiteren Kommentaren sind folgende Optionen

Pfade konfigurieren

Eigentlich sollte es reichen, nur das DEF_ROOT_DIR zu setzen e.g. /home/bierwart/bksystem. Alle anderen Pfade werden relativ dazu gesetzt

Konfiguration: einzelne scripte de/aktivieren

Die zugehörigen Variablen auf 0 setzten zum deaktivieren, 1 zum aktivieren. Die Option restore DB kann (und wäre auch sinnlos) nicht zusammen mit den anderen ausgeführt werden, also standartmäßig deaktiviert.

Kommandozeilen Argumente

Mit -h oder -help lassen sich alle Optionen anzeigen. Script aufrufen und die Optionen anhängen. Bsp:./bksystem.sh -ab -f dbfile.sqlite: alle jobs und backup von datei ...

Sub-scripte

Diese Scripte werden alle vom masterscript bksystem.sh aufgerufen. Diese Vorgehensweise ist besser, da die Environment automatisch gesetzt wird (Config, s.o.). Einzelne Scripte mit den Commanline switches (de)aktiveren

Es lassen sich aber auch alle in einer shell direkt ausführen. Dabei muss die Datenbank (mit Pfad) als export angegeben werden über die Variable DB (Pfad + Name der Datenbank e.g. export DB="/pad/zu/datenbank/name.sqlite"), oder default ist hier immer ./bksystem.sqlite. Alle Dateien werden dann ins/vom Arbeitsverzeichnis geschrieben/gelesen. Alternativ können folgende Pfadvariable gesetzt werden mit export, e.g. export BKS_DB_DIR="~/bksystem/database_dir". Crontab

Die Crontab mit crontab -e bearbeiten (i → einfügen, ESC :x ENTER → Save+Quit). Eine der folgenden Zeilen einfügen. Mehr zu Cronjobs und den Zeitangaben.

  • 0 4 1 */1 * /home/username/pfad/zum/script/bksystem.sh -a >/dev/null 2>&1 (Jeder 1. im Monat um 4:00 Uhr)
  • 0 4 * * 7 /home/username/pfad/zum/script/bksystem.sh -a >/dev/null 2>&1 (Jeden Sonntag 4:00 Uhr)
  • 0 0 */14 * * /home/username/pfad/zum/script/bksystem.sh -a >/dev/null 2>&1 (Alle 14 Tage um 0:00 Uhr)

Folgende Pfadvariable müssen angegeben sein, falls sie nicht schon gesetzt sind (einfach am Anfang der Datei einfügen):

  • PATH=/usr/local/bin:/usr/local/sbin:/sbin:/usr/sbin:/bin:/usr/bin:/usr/bin/X11
  • SHELL=/bin/sh oder SHELL=/bin/bash

WICHTIG: Leerzeile am Ende der Datei einfügen, sonst gehts nicht!
